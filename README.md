# HindiASR
Project Overview
The goal of this project is to build an accurate ASR model for Hindi grocery store dialogues, which often contain domain-specific brand names, unique measurements, and informal transaction patterns.

This repository contains three distinct stages of development:

Synthetic Data Generation & Distillation (Current): A robust, checkpoint-enabled pipeline to generate 5,000 samples of high-quality grocery audio/text pairs.

MMS-TTS Exploration (Legacy): Early attempts using Meta’s MMS-TTS model for synthetic data.

Whisper Evaluation & Scaling: Comparative study showing the impact of model size on transcription accuracy.

# Pipeline Details

1. 5k Dataset & Distillation Pipeline (5k_dataset_pipeline/)
This is the primary engine. It uses a "Teacher-Student" distillation approach to ensure high-quality training data.

Text: Generates diverse, domain-specific grocery dialogues using Google’s Gemini API.

Audio: Synthesizes audio using the Veena TTS model.

Distillation: Uses Deepgram (Nova-3) as a "Teacher" model to generate gold-standard transcripts, timestamps, and confidence scores for the synthetic audio.

Resilience: Features automatic checkpointing to handle potential Colab runtime disconnections.

2. MMS_TTS Exploration (MMS_TTS/)
Contains initial experiments using Meta's Massively Multilingual Speech (MMS) framework. While insightful for learning speech synthesis parameters, this approach was superseded by the higher fidelity and better domain control offered by the Veena architecture.

3. Whisper Evaluation (Veena_TTS_Evaluation/)
A performance analysis study comparing various Whisper model sizes (tiny to large-v3) on the synthetic audio generated by Veena.

# Key Finding: Transcription accuracy significantly increases as model parameters scale, validating the necessity of using larger models for specific domain nuances.
